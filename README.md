# contract-bot

Telegram-бот «Можно подписывать AI» для первичной AI-проверки договоров перед подписанием.

Бот принимает текст, PDF или DOCX договора, показывает бесплатный краткий риск-анализ и предлагает платный полный разбор через YooKassa.

## Что делает бот

- принимает договор текстом, PDF или DOCX;
- предлагает пользователю выбрать роль в договоре;
- отправляет текст договора в GigaChat;
- показывает FREE-результат: вердикт, score и ключевой риск;
- принимает оплату через YooKassa;
- после оплаты или промокода выдаёт PRO-разбор с рисками, последствиями и формулировками «Было → Стало».

## Основные файлы

- `bot.py` — Telegram handlers, пользовательские сценарии, обработка документов, запуск бота;
- `ai_engine.py` — prompt'ы, вызов GigaChat, парсинг и форматирование AI-ответов;
- `database.py` — SQLite-схема и операции с пользователями, анализами, платежами и промокодами;
- `payments.py` — интеграция с YooKassa;
- `config.py` — загрузка и валидация переменных окружения.

## Переменные окружения

Скопируйте пример файла окружения и заполните реальные значения:

```bash
cp .env.example .env
```

Обязательные переменные:

| Переменная | Назначение |
| --- | --- |
| `TELEGRAM_TOKEN` | Токен Telegram-бота от @BotFather |
| `GIGACHAT_TOKEN` | Токен/credentials для GigaChat |
| `YOOKASSA_SHOP_ID` | ID магазина YooKassa |
| `YOOKASSA_SECRET_KEY` | Секретный ключ YooKassa |
| `PAID_PRICE` | Цена полного анализа в рублях, по умолчанию `1490` |
| `GIGACHAT_VERIFY_SSL` | Проверка SSL для GigaChat: `false` сохраняет текущую совместимость, `true` предпочтительно после настройки сертификатов |

> Никогда не коммитьте `.env`, базу `bot.db`, логи и пользовательские документы.

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python bot.py
```

Для Windows PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env
python bot.py
```

## Деплой

Проект содержит Railway/Nixpacks-конфигурацию:

- `Procfile`
- `nixpacks.toml`

Перед деплоем убедитесь, что все обязательные переменные окружения добавлены в Railway Variables. Не храните реальные секреты в репозитории.

## Безопасность данных

Проект работает с договорами, платежами и персональными данными. Минимальные правила:

- не публиковать `.env` и значения Railway Variables;
- не коммитить `bot.db`, `bot.log`, `__pycache__` и пользовательские документы;
- не показывать токены и секреты на скриншотах;
- перед подключением новых AI-провайдеров отдельно оценивать правила передачи договоров и персональных данных.
